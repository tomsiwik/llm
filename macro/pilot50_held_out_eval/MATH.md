# Held-Out Evaluation of 50 Pilot Experts: Mathematical Framework

## Problem Statement

Given 50 distilled LoRA experts trained on synthetic domain-specific data
(generated by Llama 3.3 70B teacher), we need to determine whether the
distillation produced genuine domain knowledge transfer or merely memorized
the training distribution. The existing evaluation is contaminated: it tests
on the last 100 of 1000 training examples per domain.

## Evaluation Framework

### MMLU Log-Probability Scoring

For a multiple-choice question with prompt $p$ and answer choices
$\{A, B, C, D\}$, the model prediction is:

$$\hat{y} = \arg\max_{c \in \{A,B,C,D\}} \log P(c \mid p; \theta)$$

where $\theta$ denotes model parameters (base or base + adapter).

For adapter $i$ with parameters $\Delta\theta_i$ on MMLU subset $s$:

$$\text{acc}_i^s = \frac{1}{|D_s|} \sum_{(p,y) \in D_s} \mathbb{1}[\hat{y}(p; \theta_0 + \Delta\theta_i) = y]$$

The improvement delta is:

$$\delta_i^s = \text{acc}_i^s - \text{acc}_{\text{base}}^s$$

### Domain-to-MMLU Mapping

Let $M: \text{adapter} \to 2^{\text{MMLU subsets}}$ be the mapping function.
We have $|M| = 23$ adapters with valid MMLU mappings out of 50 total.

Per-adapter aggregate accuracy:

$$\text{acc}_i = \frac{\sum_{s \in M(i)} \text{correct}_i^s}{\sum_{s \in M(i)} |D_s|}$$

### Kill Criteria (Formal)

**K1** (win rate): $\frac{|\{i : \delta_i > 0\}|}{|M|} < 0.80 \implies \text{KILL}$

**K2** (average improvement): $\frac{1}{|M|} \sum_{i \in M} \delta_i < 0.02 \implies \text{KILL}$

**K3** (HumanEval): $\text{pass@1}(\theta_0 + \Delta\theta_{\text{python}}) < \text{pass@1}(\theta_0) \implies \text{KILL}$

### HumanEval pass@1

For HumanEval with 164 problems:

$$\text{pass@1} = \frac{1}{164} \sum_{j=1}^{164} \mathbb{1}[\text{exec}(\text{gen}(p_j; \theta)) \text{ passes } t_j]$$

where $\text{gen}$ produces a greedy completion and $\text{exec}$ runs the
test suite $t_j$ in a sandbox.

### Expected Magnitudes

**Base Qwen2.5-7B MMLU** (from literature): ~70-72% overall.
Individual subjects vary from ~30% (moral_scenarios) to ~90%+ (high_school_CS).

**Base Qwen2.5-7B HumanEval** (from literature): ~55-65% pass@1.

**Expected adapter delta**: Given training on synthetic instruction-response pairs
(not MMLU-format QA), we expect modest improvements (+1-5pp) on related MMLU
subsets. The distillation teaches domain knowledge, not test-taking skills.
A delta > 2pp averaged across 23 adapters would indicate genuine knowledge transfer.

### Statistical Considerations

MMLU test set sizes per subset range from 100 to 1534 questions.
For a subset with $n=100$, detecting a 5pp improvement (0.70 to 0.75) has:

$$\text{SE} = \sqrt{\frac{p(1-p)}{n}} \approx \sqrt{\frac{0.72 \times 0.28}{100}} \approx 0.045$$

So a 5pp delta is barely 1 SE. This means per-subset comparisons will be noisy,
but aggregate across 23 adapters (with ~40 unique MMLU subsets, ~5000+ total
questions) provides sufficient power for the kill criteria.

### Computational Cost

- Base model eval: ~40 MMLU subsets, ~200-1500 questions each.
  At ~0.5s/question on 4-bit A5000: ~30 min for base.
- Per adapter: only its mapped subsets (~2-4 subsets, ~400-800 questions).
  At ~0.5s/question: ~5 min/adapter. Loading adapter takes ~5s.
- 23 adapters total: ~115 min for adapters.
- HumanEval: 164 problems, ~2s/problem for generation + execution: ~6 min/model.
  Base + python = ~12 min.
- **Total: ~2.5 hours, ~$0.10 at $0.16/hr.**

### What This Does NOT Test

1. Multi-expert composition quality (separate hypothesis: exp_pilot50_composition_quality)
2. Cross-domain transfer (e.g., does chemistry expert help on biology?)
3. Generative quality (MMLU tests knowledge, not generation fluency)
4. Real-world task performance beyond MMLU/HumanEval
5. Scale beyond N=50
