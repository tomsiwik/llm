# MATH: P11.G0 — GRPO Refinement via RS-SFT from SFT Initialization

## Problem Statement

P11.B0 applies RS-SFT GRPO starting from the **base model** (62.1% MMLU-Pro).
P11.F0 trains a LoRA adapter via SFT on s1K traces (math-s1k-reasoning-v0).

**Question**: Can we compound the gains? Start RS-SFT GRPO from the F0 adapter
rather than from scratch, using F0's improved starting point for higher-quality rollouts.

---

## Theorem 1: SFT Initialization Improves RS-SFT Sample Efficiency

**Statement**: Let π_0 be the base model with accuracy p_0 on MMLU-Pro,
and let π_SFT = F0 adapter with accuracy p_SFT ≥ p_0. For a fixed query budget
N, the number of correct self-generated traces is:

    |D_correct(π)| = N · p(π)

where p(π) is π's accuracy on the sampling distribution.

**Lemma**: If p_SFT > p_0, then G0 (starting from π_SFT) collects more training
examples per query budget than B0 (starting from π_0):

    E[|D_correct(G0)|] = N · p_SFT ≥ N · p_0 = E[|D_correct(B0)|]

**Proof**:
The gradient variance of RS-SFT is:

    Var[∇L] ∝ 1 / |D_correct|

With more correct examples: Var[∇L_G0] ≤ Var[∇L_B0]

Lower gradient variance → smaller effective learning rate needed → more stable
convergence toward the RS-SFT fixed point.

**Furthermore**, π_SFT is closer to the GRPO fixed point than π_0 because:
- F0 already learned to produce correct reasoning for some MMLU-Pro questions
- The RL refinement phase has less "exploratory noise" to overcome
- Theorem 3 of B0 (self-generated traces are in-distribution) holds equally for
  traces generated by π_SFT

**QED.**

---

## Theorem 2: Non-Regression Guarantee (D_train = D_eval)

**Statement**: G0 trains on self-generated correct MMLU-Pro traces (D_train = MMLU-Pro).
By Theorem 1 of B0 (EWC, Kirkpatrick 2017, arXiv:1612.00796):

    If D_train = D_eval: ∀ gradient step that reduces L(θ, D_train),
    E[ΔL(θ, D_eval)] ≤ 0

**Consequence**: G0 cannot catastrophically forget MMLU-Pro by construction.
This is the same guarantee as B0, but G0 starts from a better θ_init.

**Prediction**: G0 accuracy on MMLU-Pro ≥ p_SFT (no regression from F0 adapter).

**QED.**

---

## Theorem 3: RS-SFT Approximates GRPO (inherited from B0)

**Reference**: B0 MATH.md, Theorem 2 (Shao et al. 2024, arXiv:2402.03300).

RS-SFT = GRPO with:
1. Only positive-advantage terms (correct completions)
2. Advantage weight absorbed into learning rate
3. Reference policy correction omitted (LoRA = small update, KL ≈ 0)

For G0: π_SFT serves as both the rollout policy AND the reference policy.
The KL penalty term in full GRPO: KL(π_θ || π_SFT) ≈ 0 at initialization,
which is exactly the RS-SFT assumption. This is tighter than B0 where
KL(π_θ || π_0) also ≈ 0 but π_0 is further from the RL fixed point.

## Kill Criteria

- **K1514**: G0 adapter >= 70% MMLU-Pro with thinking (beats Google 69.4%)
  Pass = compound SFT+RL gains exceed individual contributions
- **K1515**: G0 GSM8K >= F0 GSM8K (no regression from F0 adapter)
  Pass = Theorem 2 confirmed (D_train=MMLU-Pro does not harm GSM8K)
- **K1516**: G0 >= F0 + 3pp on at least one benchmark
  Pass = RL refinement adds measurable signal above SFT-only baseline

## Failure Modes

1. **F0 accuracy = base (62.1%)**: If F0 doesn't improve over base, G0 has same
   yield as B0. Theorem 1 gives G0 ≈ B0. K1514 and K1516 may fail.

2. **MMLU-Pro RS-SFT plateau**: If 62-65% is the RS-SFT ceiling for this model
   size, both B0 and G0 will plateau. The 70% target may be unreachable.
   Kill: K1514 fails. Finding: RS-SFT ceiling identified, propose GRPO+KL.

3. **GSM8K regression**: Training on MMLU-Pro traces might reduce GSM8K performance.
   Theorem 2 prevents MMLU-Pro regression but NOT cross-domain regression.
   Kill: K1515 fails.

## Algorithm

```
Phase 1: Rejection Sampling with F0 Adapter
  Load model + adapters/math-s1k-reasoning-v0/
  For each of N_SAMPLE_QUESTIONS MMLU-Pro questions (stratified, seed=42):
    Generate response with thinking=True (greedy, default)
    Keep if correct → training dataset

Phase 2: SFT Training from F0 Initialization
  mlx_lm.lora --resume-adapter-file .../math-s1k-reasoning-v0/adapters.safetensors
  Training on self-generated MMLU-Pro traces
  Save to adapters/math-s1k-grpo-v0/

Phase 3: Three-Way Evaluation
  Eval base: 280q MMLU-Pro + 50q GSM8K
  Eval F0: same
  Eval G0: same
```
