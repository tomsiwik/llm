# MATH.md: Cross-Domain M2P Graph with Dissolve-Recrystallize Cycle

## TYPE: guided-exploration
## PROVEN FRAMEWORK: Grassmannian orthogonality (Finding #3, #341 K848)
## UNKNOWN: Whether cross-domain M2P adapters provide useful transfer, and whether dissolving them into the base improves per-domain quality

---

## Setting

We have N domains. Per-domain M2P generates adapters at 93.3% of SFT quality (Finding #351).
The question: can we do better by also learning **cross-domain transfer knowledge**?

### Definitions

**Per-domain adapter:** Δ_d = B_d A_d, where A_d is a frozen Grassmannian slot and B_d is generated by M2P_d from domain d context. These are proven to work (#351).

**Cross-domain adapter:** Δ_{a→b} = B_{a→b} A_{a→b}, where A_{a→b} is a SEPARATE frozen Grassmannian slot and B_{a→b} is generated by a cross-domain M2P. This adapter captures "what domain a knows that helps domain b."

**Grassmannian capacity:** For N domains with C cross-domain pairs, we need N + C total slots. Each slot needs `rank` orthogonal columns in ℝ^d. Capacity: (N + C) × rank ≤ d.
- At d=256, rank=4, N=5: C ≤ (256/4) - 5 = 59 pairs. We test C=10 (all pairs). ✓

### Orthogonality Guarantee

**Theorem 1 (unchanged):** All adapters — per-domain AND cross-domain — occupy distinct Grassmannian A-slots. By QR construction:

A_i^T A_j = 0  for all i ≠ j (including cross-domain slots)

Therefore:

⟨Δ_i, Δ_j⟩_F = trace(B_j (A_j A_i^T) B_i^T) = 0

**This holds for ANY combination of per-domain and cross-domain adapters.** No parameter interference regardless of how many cross-domain links we add.

## Three Training Options

### Option A: Cross-Prediction

Train cross-domain M2P_{a→b} to generate adapters that help domain b, using domain a as context.

**Loss:**
```
L_A(a→b) = CE(base + Δ_{a→b}, tokens_b)
```

where Δ_{a→b} uses A-slot for pair (a,b) and B generated from domain a context.

**Intuition:** "What does understanding domain a contribute to domain b?"
**Example:** Showing M2P arithmetic patterns, asking it to help with sort.

**Prediction:** Cross-prediction loss < base loss on domain b IFF domains share transferable structure. For synthetic domains (arithmetic↔parity share number understanding), expect >10% improvement. For unrelated pairs (repeat↔sort), expect ~0%.

### Option B: Residual Transfer

First obtain per-domain adapters (proven at 93.3%). Then train cross-domain M2P on what domain b STILL gets wrong.

**Loss:**
```
L_B(a→b) = CE(base + Δ_b + Δ_{a→b}, tokens_b^{hard})
```

where tokens_b^{hard} are examples where per-domain adapter Δ_b has high loss (residual errors).

**Intuition:** "What does domain a know that domain b is STILL missing?"
**Advantage:** Doesn't duplicate within-domain knowledge. Learns only the transfer gap.
**Risk:** Residual errors may be irreducible (base model limitations, not missing knowledge).

### Option C: Combined (Cross-Prediction + Residual)

Joint loss:
```
L_C(a→b) = α · CE(base + Δ_{a→b}, tokens_b) + (1-α) · CE(base + Δ_b + Δ_{a→b}, tokens_b^{hard})
```

with α=0.5 (equal weight).

**Intuition:** Learn both general transfer AND specific gap-filling.

## The Dissolve-Recrystallize Cycle

After training all cross-domain adapters, we run a promotion cycle:

### Step 1: Compose
Apply all cross-domain adapters to the base simultaneously:
```
h_enriched = W_base·x + Σ_{(a,b)} scale · B_{a→b}(A_{a→b}·x)
```

### Step 2: Dissolve (Promote)
Merge cross-domain adapters into the base weights at scale=5 (proven safe: Finding #333):
```
W_enriched = W_base + scale · Σ_{(a,b)} A_{a→b} B_{a→b}^T
```

Note: this is a LoRA merge, not a forward pass. The Grassmannian A-slots for cross-domain adapters are now "spent" (absorbed into base), but the per-domain A-slots are untouched.

### Step 3: Recrystallize
Retrain per-domain M2P on the enriched base. The enriched base already "knows" cross-domain relationships, so per-domain adapters should:
- Train faster (base is closer to target)
- Achieve higher quality (base provides better features)

### Conjecture 2: Enrichment Monotonicity — REFUTED

**Status: REFUTED by experiment.** Renamed from "Theorem 2" because (a) the proof sketch is circular and (b) the prediction was empirically falsified.

**Claim:** If cross-domain adapters reduce per-domain loss on their target domains, then the enriched base provides a strictly better starting point for per-domain adaptation.

**Proof sketch (CIRCULAR — see note):**
Let L_d(W) = per-domain loss on domain d using base W.
After promotion: W' = W + Σ scale · ΔW_{cross}.
If the cross-domain adapters were useful: L_d(W') ≤ L_d(W) for most d.
Per-domain M2P trained on W' starts from a lower base loss → achieves lower final loss
(assuming M2P quality ratio is roughly constant, lower base loss → lower adapted loss).

**Circularity note:** The hidden assumption "M2P quality ratio is roughly constant" is exactly what the conjecture claims to prove. The argument reduces to: if adapters are useful and quality ratio is preserved, then quality improves — which is tautological. This is NOT a proof.

**Empirical refutation:** The parity enriched base loss increased from 0.59 to 3.73 (6.3x worse), directly contradicting the monotonicity claim. The root cause: 10 cross-domain adapters merged at PROMOTE_SCALE=5 while trained at LORA_SCALE=2 produces 2.5x amplification per adapter, or 25x total effective scale, far exceeding what the dissolve step was validated for (single-adapter, Finding #333). A valid theorem would need to bound the maximum harm from merging N adapters simultaneously.

**Prediction (superseded by refutation):** Per-domain M2P quality on enriched base ≥ quality on original base.
Quantitative: expect ≥5% improvement in quality ratio for domains with strong cross-domain signal.

### Conjecture 3: Grassmannian Slot Recycling — UNSUBSTANTIATED

**Status: UNSUBSTANTIATED.** Renamed from "Theorem 3" because no proof is provided and the cycle repetition is not tested in the experiment.

After dissolve, the cross-domain A-slots are "spent" — their information is in the base weights. But the A-slots themselves can be REUSED for the next cycle of cross-domain M2P, because:

1. New QR decomposition generates fresh orthogonal A-slots
2. These are orthogonal to the per-domain A-slots (which are unchanged)
3. The next cycle's cross-domain adapters learn the RESIDUAL cross-domain transfer (what the first cycle missed)

**Note:** Point 1 is true by construction. Points 2 and 3 are assertions without proof. In particular, the claim that subsequent cycles learn "residual" transfer (rather than re-learning similar adapters) is unsubstantiated and untested. This conjecture is a plausible direction for future work, not an established result.

## Directional Asymmetry Limitation

The experiment uses `cross_slot_map[(a,b)] = cross_slot_map[(b,a)]`, mapping both directions of a pair to the same Grassmannian slot. Training runs only over `combinations(range(5), 2)`, which generates pairs where `a < b` by index. This means:

- 10 of 20 possible directed pairs are tested (unidirectional: a→b where a < b by index)
- The reverse direction (b→a) is NOT trained — e.g., "sort→arithmetic" is not tested even though "arithmetic→sort" is
- The shared slot means that even if both directions were trained, they would share the same adapter geometry, preventing per-direction specialization

This is a design choice that reduces the slot count requirement (10 instead of 20), but it means the question "does arithmetic help sort?" and "does sort help arithmetic?" are treated as interchangeable. They may not be. Directional asymmetry in transfer is untested.

## Kill Criteria

**K_A: Cross-domain adapter quality.**
At least 3/10 cross-domain pairs must reduce target-domain loss by >5% vs base alone.
(If no pair helps, cross-domain transfer doesn't exist in our synthetic domains.)

**K_B: Enriched base improvement.**
Per-domain M2P quality ratio on enriched base must be ≥ quality on original base for ≥3/5 domains.
(If enrichment doesn't help, the dissolve cycle is pointless.)

**K_C: Grassmannian preservation.**
All adapter pairs (per-domain AND cross-domain) must have |cos| < 1e-5.
(Structural guarantee — should pass by construction.)

**K_D: Option comparison.**
The best option (A, B, or C) must achieve >50% quality ratio (median across domains) on the enriched base.
(If none exceeds 50%, the architecture improvement is marginal.)

## Self-Test (MANDATORY — answer before running experiment)

1. **What is the ONE mathematical property that makes the failure mode impossible?**
   Answer: Theorem 1 only. Grassmannian QR orthogonality makes parameter-space interference between any two adapters (per-domain or cross-domain) geometrically impossible — their A-slots are by construction orthogonal, so Frobenius inner product is zero. There is NO proven property that makes quality regression after dissolve impossible; Conjecture 2 was refuted.

2. **Which existing theorem(s) does the proof build on?**
   - Theorem 1 builds on Finding #3 / K848 (Grassmannian orthogonality via QR decomposition, proven in prior experiments). The QR construction is classical linear algebra (Gram-Schmidt). No new theorem is proven here beyond re-applying the existing result to an extended slot set.
   - Conjecture 2 (Enrichment Monotonicity) was NOT grounded in any existing theorem. It assumed its conclusion (circular) and was empirically refuted.
   - Conjecture 3 (Slot Recycling) is grounded in QR construction for point 1, but points 2-3 have no theorem basis.

3. **What specific numbers does the proof predict?**
   From Theorem 1: |cos(A_i, A_j)| = 0 for all i ≠ j, measured as < 1e-8 in practice (machine precision, verified at 1.02e-08).
   From Conjecture 2 (refuted): ≥5% quality improvement for related domains. ACTUAL: parity 0.59 → 3.73 (6.3x regression).
   From cross-domain transfer analysis: 3-5/10 useful pairs predicted. ACTUAL: 8/10 useful.
   See PAPER.md Prediction vs. Measurement table for full comparison.

4. **What would FALSIFY the proof (not just the experiment)?**
   Theorem 1 is falsified if |cos(A_i, A_j)| > machine epsilon for any QR-constructed pair — this would require a bug in the QR construction or numerical instability at larger d.
   Conjecture 2 was already falsified: the parity enriched base loss increased 6.3x, demonstrating that dissolving 10 adapters at PROMOTE_SCALE=5 does NOT guarantee monotone improvement.

5. **How many hyperparameters does this approach add?**
   Count: 5 (PROMOTE_SCALE=5, LORA_SCALE=2, alpha=0.5, CROSS_STEPS=300, M2P_LR=1e-3).
   - PROMOTE_SCALE=5: carried over from Finding #333 (single-adapter validation), not re-derived for N=10 adapters. Should be PROMOTE_SCALE / N for safety.
   - LORA_SCALE=2: training-time scale, determined empirically in prior work.
   - alpha=0.5: mixing weight for Option C, unexplored (only 0.5 was tested).
   - CROSS_STEPS=300: training budget for cross-domain M2P, set by analogy to per-domain training.
   - M2P_LR=1e-3: learning rate, standard default, not derived.
   All 5 are unjustified by theory in this context.

6. **Hack check: Am I adding fix #N to an existing stack?**
   This is a guided exploration (fix #1 on top of the M2P + Grassmannian foundation), which is acceptable. The dissolve-recrystallize cycle is one mechanism on top of a proven orthogonality foundation. However, three options (A, B, C) plus a dissolve step plus a quality ratio metric represents accumulated complexity. The single underlying question — "does cross-domain knowledge help per-domain M2P?" — could have been tested with just Option A. Options B and C, while not harmful, are scope creep relative to the core question.
