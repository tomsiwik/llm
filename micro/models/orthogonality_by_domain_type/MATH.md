# Orthogonality by Domain Type: Mathematical Foundations

## 1. Setup and Notation

| Symbol | Definition | Shape/Value |
|--------|-----------|-------------|
| d | Model embedding dimension | 64 (micro) |
| d_ff | MLP intermediate dimension | 256 (= 4d) |
| L | Number of MLP layers | 4 |
| r | LoRA rank | 8 |
| alpha | LoRA scaling factor | 8 |
| N | Number of domain experts | 15 |
| K | Number of semantic clusters | 3 |
| M | Domains per cluster | 5 |
| V | Vocabulary size | 32 |
| T | Context length | 16 |

## 2. LoRA Delta Structure

For each MLP layer l, we have two LoRA-adapted linear maps:

    fc1_adapted(x) = x @ W1_l + x @ A1_l @ B1_l * (alpha/r)
    fc2_adapted(x) = x @ W2_l + x @ A2_l @ B2_l * (alpha/r)

where:
- A1_l: (d, r), B1_l: (r, d_ff) -- fc1 LoRA
- A2_l: (d_ff, r), B2_l: (r, d) -- fc2 LoRA

The weight deltas are:

    dW1_l = (alpha/r) * A1_l @ B1_l    shape: (d, d_ff)
    dW2_l = (alpha/r) * A2_l @ B2_l    shape: (d_ff, d)

## 3. Delta Vector Construction

The full delta vector for expert i is the flattened concatenation of all
per-layer deltas:

    v_i = concat[
        vec(A1_0 @ B1_0), vec(A2_0 @ B2_0),
        vec(A1_1 @ B1_1), vec(A2_1 @ B2_1),
        ...,
        vec(A1_{L-1} @ B1_{L-1}), vec(A2_{L-1} @ B2_{L-1})
    ]

Dimension of v_i:

    D = L * (d * d_ff + d_ff * d) = L * 2 * d * d_ff

For micro scale:

    D = 4 * 2 * 64 * 256 = 131,072

This is the space in which we measure pairwise cosine similarity.

## 4. Cluster Model

### 4.1 Data Generation

Domain data is generated by Markov chains on the character vocabulary.
Each cluster c has a prototype transition matrix P_c: (V, V), where
P_c[i, j] = Pr(next_token = j | prev_token = i).

Domain d in cluster c has transition matrix:

    T_d = softmax(log(P_c) + eta_d + epsilon)

where:
- eta_d ~ N(0, sigma_domain^2) is a domain-specific perturbation
- epsilon ~ N(0, sigma_noise^2) is sampling noise
- sigma_domain = 0.075 (half of noise_scale * 0.5)
- sigma_noise = 0.15

### 4.2 Expected Similarity Structure

Two domains d1, d2 in the same cluster c share the prototype P_c,
differing only by their perturbations eta_d1, eta_d2. The resulting
data distributions p(x|d1) and p(x|d2) are close in KL divergence:

    KL(p(x|d1) || p(x|d2)) ~ O(sigma_domain^2)

Two domains in different clusters have fundamentally different prototypes,
so:

    KL(p(x|d1) || p(x|d2)) ~ O(sigma_cluster^2 + sigma_domain^2)

where sigma_cluster = cross_cluster_distance = 2.0, much larger than
sigma_domain = 0.075.

### 4.3 Gradient Flow Creates Similarity

LoRA training via SGD on data from domain d produces gradients:

    grad_B1_l = (alpha/r) * sum_{(x,y) in batch} (x @ A1_l)^T @ d_z1

where d_z1 is the gradient of the loss w.r.t. the fc1 pre-activation.

For domains in the same cluster, the data distributions p(x|d) are
similar, producing similar gradients. After T steps of SGD:

    B1_l(d) ~ B1_l(0) - lr * sum_{t=1}^{T} grad_B1_l(d, t)

The accumulated gradient sum determines B1_l(d), and similar data
distributions produce correlated gradient sums.

## 5. Expected Cosine by Group

### 5.1 Random Baseline

For random vectors in D=131,072 dimensions:

    E[|cos(v_i, v_j)|] = sqrt(2 / (pi * D)) = sqrt(2 / (pi * 131072))
                        = 0.00220

### 5.2 Within-Cluster Expectation

Domains within a cluster share a data prototype. Their LoRA deltas v_i, v_j
have a shared component (from the common prototype) plus independent noise
(from domain-specific perturbations):

    v_i = v_shared_c + delta_i
    v_j = v_shared_c + delta_j

where v_shared_c is the delta that any domain in cluster c would produce,
and delta_i, delta_j are independent perturbations.

    cos(v_i, v_j) = (||v_shared||^2 + v_shared . (delta_i + delta_j) + delta_i . delta_j)
                    / (||v_i|| * ||v_j||)

When ||v_shared|| >> ||delta_i||:

    cos(v_i, v_j) ~ ||v_shared||^2 / ||v_i||^2 ~ 1 - O(||delta||^2/||v_shared||^2)

When ||delta_i|| ~ ||v_shared||:

    cos(v_i, v_j) ~ 0.5 (moderate similarity)

### 5.3 Cross-Cluster Expectation

For domains in different clusters, v_shared_c1 and v_shared_c2 are
independent (different prototypes), so:

    cos(v_i, v_j) ~ cos(v_shared_c1, v_shared_c2) + noise

Since the cluster prototypes are drawn independently with
cross_cluster_distance = 2.0:

    E[|cos(v_shared_c1, v_shared_c2)|] ~ sqrt(2 / (pi * D))

This is the random baseline, meaning cross-cluster pairs should have
cosine near random chance.

## 6. Statistical Testing

### 6.1 Pair Counts

With N=15 domains and K=3 clusters of M=5:
- Total pairs: C(15, 2) = 105
- Within-cluster pairs: K * C(M, 2) = 3 * 10 = 30
- Cross-cluster pairs: 105 - 30 = 75

### 6.2 Permutation Test

H0: Within-cluster and cross-cluster cosines come from the same distribution.
H1: Within-cluster cosines are stochastically larger.

Test statistic: T = mean(|cos|_within) - mean(|cos|_cross)

Under H0, any assignment of cosine values to the within/cross partition
is equally likely. We estimate the null distribution by randomly permuting
the within/cross labels 10,000 times:

    p-value = Pr(T_perm >= T_observed | H0)
            = (1/10000) * sum_{perm} I(T_perm >= T_observed)

### 6.3 Effect Size

Cohen's d measures the standardized difference:

    d = (mean_within - mean_cross) / s_pooled

where s_pooled = sqrt((s_within^2 + s_cross^2) / 2).

Interpretation:
- d < 0.2: negligible
- 0.2 <= d < 0.5: small
- 0.5 <= d < 0.8: medium
- d >= 0.8: large

## 7. Assumptions

1. **Markov chain data is a valid proxy for domain similarity.** Real
   domains differ in complex ways (syntax, semantics, reasoning patterns).
   Our Markov chains capture the simplest form of distributional similarity
   (character n-gram statistics). If the effect holds for simple
   distributional similarity, it should be stronger for richer semantic
   similarity (more correlated gradients).

2. **MLP model suffices to test LoRA orthogonality.** We use a 4-layer
   MLP instead of a transformer. Since LoRA is applied only to MLP layers
   (FFN-only, per project findings), the attention mechanism is not
   relevant to the LoRA delta structure. The MLP forward/backward dynamics
   through the LoRA parameters are identical.

3. **Approximate backprop is sufficient.** We propagate gradients only
   through the local MLP block (not through the residual connection to
   earlier layers). This is a standard approximation for LoRA training
   since the base model is frozen and LoRA is a small perturbation.

4. **B-only training is valid.** We freeze A (random projection) and
   train only B, following standard LoRA practice where A provides a
   random subspace and B learns the task-specific linear map within
   that subspace.

## 8. Worked Example

d=64, d_ff=256, L=4, r=8, N=15, K=3, M=5:

    Delta vector dimension: D = 4 * 2 * 64 * 256 = 131,072
    Random E[|cos|]: sqrt(2 / (pi * 131072)) = 0.00220

    N_max (orthogonality capacity): D / r^2 = 131072 / 64 = 2,048

    Within-cluster pairs: 3 * C(5,2) = 30
    Cross-cluster pairs: C(15,2) - 30 = 75

Observed (3-seed average):
    Within-cluster mean |cos|: 0.060
    Cross-cluster mean |cos|:  0.008
    Ratio: 7.84x
    Cohen's d: 2.24

Both within and cross are above the random baseline (0.002), confirming
that training (even minimally) creates structure. But within is 7.84x
higher than cross, showing that cluster membership dominates.

## 9. Scaling Considerations

At macro scale (Qwen2.5-7B, d=3584, d_ff=18944, L=28, r=16):

    D_FFN = 28 * (3584 * 18944 + 18944 * 3584) = 3.80 billion
    Random E[|cos|]: sqrt(2 / (pi * 3.8e9)) = 9.2e-6

    Within-cluster cosine at macro: expected to decrease in absolute
    value (higher D -> lower cosine for random components) but the
    RATIO (within/cross) should remain or increase, because the shared
    cluster signal is proportionally amplified by more expressive models
    that can better separate domain-specific features.

    Key prediction: at d=3584, within-cluster |cos| will be much smaller
    in absolute terms (~10^{-4} to 10^{-3}) but still significantly
    higher than cross-cluster (~10^{-5} to 10^{-4}).
