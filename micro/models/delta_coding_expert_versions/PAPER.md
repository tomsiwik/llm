# Delta Coding for Expert Version Management: Research Digest

## Hypothesis

Expert version updates can be stored as deltas (like video P-frames) with
<1% quality drift per delta application and <50% storage cost vs full snapshots.

**Falsifiable**: If quality drifts >1% per delta application or if delta storage
exceeds 50% of full expert storage, the hypothesis is killed.

---

## What This Model Is

A version management system for LoRA experts inspired by video compression.
When an expert evolves (v1 -> v2 -> v3 -> ...), instead of storing full
parameter snapshots at each version, we store:

1. **Keyframes** (I-frames): Full parameter snapshots every K versions
2. **Deltas** (P-frames): Compressed inter-version differences

Any version is reconstructed by: v_t = keyframe + sum(compressed_deltas).

The deltas are compressed using truncated SVD, exploiting the observation
that incremental fine-tuning updates lie in a low-rank subspace.

**Connection to LoRA**: LoRA itself is delta coding (dW = A@B is a compressed
delta from base weights). This work extends delta coding to the temporal
dimension -- compressing the CHANGES between consecutive LoRA versions.

## Key References

- **Video Compression (H.264/MPEG)**: I-frame/P-frame/GOP structure for
  temporal redundancy exploitation. The direct inspiration for this work.
- **LoRA** (Hu et al., ICLR 2022): Low-rank adaptation as spatial delta coding.
- **DARE** (Yu et al., 2023): Delta parameter redundancy -- up to 99% of
  fine-tuning deltas can be dropped. Supports our claim that deltas are
  highly compressible.
- **lora_procrustes** (our prior work): Confirmed LoRA deltas are pure linear,
  enabling exact weight-space decomposition.

## Empirical Results

### Setup

- d=64, r=8 (LoRA rank), L=4 layers, 20,480 LoRA params per version
- 5 expert versions generated by sequential fine-tuning on different data subsets
- Protocol: pretrain base (200 steps) -> LoRA v1 (200 steps) -> v2..v5 (80 steps each)
- 3 seeds (42, 123, 7)

### Raw Delta Coding (No Compression)

| Metric | Value | Notes |
|--------|-------|-------|
| Reconstruction error | ~1.6e-08 | Floating-point precision |
| Quality drift (4 chained deltas) | 0.0000% | Exact reconstruction |
| Storage ratio | 100% | Raw deltas = same size as full params |

Raw delta coding provides EXACT reconstruction but NO storage savings.
The savings come from compression.

### Compressed Delta Coding (SVD Truncation, K=5)

| SVD Rank | Delta Compression | Storage Ratio | Savings | Max Drift (3 seeds) | Status |
|----------|-------------------|---------------|---------|---------------------|--------|
| 1 | 13.2% | 30.6% | 69.4% | 1.722% | drift KILLS |
| **2** | **26.4%** | **41.1%** | **58.9%** | **0.796%** | **PASS** |
| 4 | 52.8% | 62.3% | 37.7% | 0.194% | storage KILLS |
| 8 (full) | 100% | 100% | 0% | 0.000% | no savings |

**SVD rank 2 is the optimal operating point**: 58.9% storage savings with
<1% quality drift across all seeds and chain positions.

### Inter-Version Delta Analysis

| Metric | Mean | Range |
|--------|------|-------|
| ||delta|| / ||params|| | 0.371 | 0.312 - 0.440 |
| SVD rank-2 relative error | 0.571 | 0.525 - 0.603 |

Deltas are ~37% of param norm (not trivially small), but SVD rank 2 captures
~43% of delta energy -- enough for <1% function-space drift.

### Kill Criteria

| Criterion | Value | Threshold | Result |
|-----------|-------|-----------|--------|
| KC1: Quality drift (raw) | 0.0000% | <1% | **PASS** |
| KC1: Quality drift (SVD r=2) | 0.796% | <1% | **PASS** |
| KC2: Storage ratio (SVD r=2) | 41.1% | <50% | **PASS** |

## Analysis

### 1. Raw Delta Coding Is Trivially Exact

Because LoRA parameters are plain arrays (no quantization, no nonlinearity),
delta reconstruction via addition is exact to floating-point precision.
This is the strongest possible result for KC1 -- zero drift with raw deltas.

The challenge is entirely in COMPRESSION, not in the delta mechanism itself.

### 2. SVD Rank 2 Is the Sweet Spot

The quality-storage tradeoff has a clear knee at SVD rank 2:
- Rank 1: too aggressive (1.7% drift on worst seed), but 69% savings
- Rank 2: safe (<0.8% drift), 59% savings
- Rank 4: excessive quality (0.2% drift) but only 38% savings

This matches intuition: inter-version deltas from short fine-tuning runs
(80 steps) are approximately rank-2 because the gradient updates during
such short training occupy a very low-dimensional subspace.

### 3. Chain Length Matters for Compressed Deltas

With SVD rank 2 compression:
- v3 (2 chained deltas): ~0.3% drift
- v5 (4 chained deltas): ~0.5% drift

Drift scales roughly linearly with chain length, as expected from additive
error accumulation. At chain length 4, we are still well within the 1% budget.
Keyframe intervals K=5 to K=10 are practical. Beyond that, periodic keyframes
(analogous to GOP structure in video) prevent unbounded drift.

### 4. Connection to DARE

DARE showed that 90%+ of fine-tuning delta parameters can be dropped without
quality loss. Our SVD compression achieves similar compression (87% at rank 1,
74% at rank 2) but on INTER-VERSION deltas rather than base-to-adapted deltas.
The mechanisms are complementary: DARE compresses spatial deltas, SVD compresses
temporal deltas.

### 5. Practical Implications for the Contribution Protocol

For the expert registry (VISION.md), delta coding enables:
- **Version history at 41% storage cost**: store 5 versions for the price of ~2
- **Rollback capability**: reconstruct any previous version from keyframe + deltas
- **Incremental updates**: contributors upload only the compressed delta, not full params
- **At scale**: 1000 experts with 10 versions each = 10K snapshots.
  Full storage: 10K * P. Delta-coded: ~4.1K * P (59% savings)

## Micro-Scale Limitations

1. **LoRA rank 8 is very low.** At macro scale (rank 64+), deltas between
   consecutive versions may have higher effective rank, changing the optimal
   SVD truncation rank. The 37% delta/param ratio may also change.

2. **80-step fine-tuning is short.** Real expert updates may involve thousands
   of steps, producing higher-rank deltas that are less compressible.

3. **Character-level names data.** With real domains, the structure of
   inter-version changes could differ substantially.

4. **5 versions is a short chain.** Production systems might have 50+ versions.
   Periodic keyframes (K=10 or K=20) would need validation at longer chains.

5. **3 seeds.** Directional evidence only. The 1.722% drift at SVD rank 1
   (seed 7) shows sensitivity to the specific update trajectory.

6. **SVD is one compression method.** Learned compression (autoencoder),
   quantization, or structured pruning of deltas could perform differently.

---

## What Would Kill This

### At Micro Scale
- Already partially survived: both kill criteria pass at SVD rank 2.
- Would be killed if more seeds showed rank-2 drift exceeding 1%.

### At Macro Scale
- **High-rank deltas**: If macro-scale expert updates have effective rank >> 8,
  SVD truncation to rank 2 would destroy too much information.
- **Quantized weights**: If LoRA params are stored in int4/int8, delta coding
  introduces quantization noise that accumulates differently.
- **Non-sequential updates**: If expert versions are not sequential (branch/merge),
  the linear chain assumption breaks and more complex version DAGs are needed.
- **Update step count**: If real expert updates use 10K+ steps (vs 80 here),
  the delta/param ratio could approach 1.0, eliminating compressibility.

---

## Key Takeaways

1. **Raw delta coding is exact** for LoRA parameters. Zero quality drift.
   The mechanism is trivially correct -- it's just addition.

2. **SVD rank 2 compression** achieves 41.1% storage ratio with <0.8% max
   drift across 3 seeds and 4 chained deltas. This is the Pareto-optimal
   operating point.

3. **The video codec analogy holds**: keyframe interval K=5 with rank-2
   SVD-compressed deltas is practical. Longer chains need periodic keyframes
   to bound accumulated drift.

4. **Delta coding + LoRA = two levels of compression**: LoRA compresses
   base-to-adapted (spatial), SVD compresses version-to-version (temporal).
   They compose without interaction.

5. **Status: PASS.** Both kill criteria met at SVD rank 2.
