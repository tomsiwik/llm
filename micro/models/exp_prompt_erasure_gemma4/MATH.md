# MATH.md — exp_prompt_erasure_gemma4

## Identification
- **Experiment**: Orca 2 Prompt Erasure on Gemma-4-E4B-it-4bit.
- **Question**: Does the Explanation-Tuning + Prompt-Erasure recipe (Mitra et al. 2311.11045 §4.2) install a reasoning *method* into the model such that the method is invoked without an explicit system-prompt trigger, while leaving background knowledge untouched?
- **Type**: verification (recipe is published; we test whether it transfers to Gemma-4).

## Prior work
- **Orca 2** (Mitra et al., arxiv:2311.11045). Trains a student on rationale traces generated by a teacher under a method-inducing system prompt; at train time the system prompt is *erased*, so the student must learn to invoke the method from the user turn alone. Reports stronger method use than Mistral-7B baseline with no explicit trigger.
- **Finding #536** (this repo): MCQ adapter training on Gemma 4 destroys native thinking mode by −11.7pp on MMLU-Pro + thinking.
- **exp_knowledge_disentanglement_control** (sibling, completed 2026-04-18, PROVISIONAL / db=killed; smoke N=25 × 60 steps). Implemented exactly the Orca-2 recipe: teacher = METHOD_SYSTEM_PROMPT with numbered subgoals (Restate → Identify → Evaluate → Pick) + `enable_thinking=True`; student trained on `{user, assistant}` pairs (system prompt erased). Adapter rank-16 on `v_proj+o_proj`, scale 4.0, top-16 layers. Evaluated WITHOUT any system prompt (prompt-erased inference).
- **Ke et al., arxiv:2302.01107** — full fine-tune method transfer baseline (unrelated architecture; noted).

## Failure modes we are preventing
1. **F1 — Trigger dependence**: adapter learns the method only when the method prompt is present; disappears under prompt erasure.
2. **F2 — Knowledge collapse**: method adapter degrades factual recall / general-knowledge scores on held-out benchmarks (MMLU-Pro).
3. **F3 — Untrained method leakage**: base already emits numbered-step structure in thinking mode → apparent method invocation is noise, not learned behaviour.

## Theorem (informal)
Let `A` be a rank-`r` LoRA adapter trained on the prompt-erased objective
    `L(θ_A) = E_(x,y~D_student) [ - log p(y | x; θ_base + θ_A) ]`
where `D_student = {(user_q, strip_thinking(teacher_response(METHOD_PROMPT, user_q)))}`. Then if training reaches loss ≤ 0.7 and the adapter is capacity-sufficient:

**Prediction P1 (K1721)** — Adapter invokes the *trained method signature* (specific lexical markers "Restate", "Identify", "Evaluate", "Pick", or the "Step N:" / "Thinking Process:" / numbered-list template matching 3 of 4 phases) on ≥50% of held-out MCQ queries posed WITHOUT the method prompt, **and** at a rate ≥ base + 20pp (absolute delta beyond base's native numbered-step behaviour).

**Prediction P2 (K1722)** — Adapter's MMLU-Pro (or cais/mmlu at smoke) accuracy stays within ±2pp of base under prompt-erased inference; the adapter encodes a *dispositional method*, not a content-replacement.

**Prediction P3 (K1723)** — Recipe fidelity: teacher trace was generated with METHOD_PROMPT + `enable_thinking=True`; student training messages contain no `system` role; student inference contains no `system` role. Three structural assertions on the training pipeline.

## Proof sketch
P1 follows from gradient-descent on `L(θ_A)` reducing KL to the teacher distribution `p(y | user, METHOD_PROMPT)`; under prompt-erased inputs the only way to reduce loss is to internalise the prompt-induced method as a function of `user_q` alone (Mitra et al. Lemma 1 equivalent: the student marginal over system-prompt matches the teacher conditional when |D_student| → ∞). At finite sample size the bound loosens; we require ≥ +20pp over base to exceed the native numbered-step prior of Gemma-4 thinking mode (F3 guard).

P2 follows from LoRA's low-rank update preserving the bulk of `θ_base`; if method injection is orthogonal to factual-recall circuits, `|Δ_MMLU| < 2pp`. If Δ exceeds 2pp, method training is **not** disentangled from knowledge — the trained signal bleeds into content.

P3 is structural and checked by inspecting training artifacts.

## Kill criteria (pre-registered, locked before run)
- **K1721** (DB id 1721) — Method invocation rate ≥ 50% on held-out MMLU/MCQ queries without system prompt, AND rate ≥ base_rate + 20pp.
- **K1722** (DB id 1722) — |MMLU accuracy delta| ≤ 2pp (adapter vs base, prompt-erased inference).
- **K1723** (DB id 1723) — Recipe fidelity: (a) teacher prompt = METHOD_PROMPT, (b) student messages contain no `system` role, (c) student inference contains no `system` role. All 3 must hold.

## Assumptions (A1–A5)
- **A1** — `exp_knowledge_disentanglement_control` adapter is a faithful Orca-2 recipe realisation on Gemma-4. Verified by reading sibling's `run_experiment.py`: teacher uses `METHOD_SYSTEM_PROMPT` with 4-step pattern; training `messages` = `{user, assistant}` only; eval chat template has no `system` role.
- **A2** — Method signature = explicit lexical markers from METHOD_PROMPT plus the numbered-step template. Regex: `(?i)(restate|identify.*(information|relevant)|evaluate.*option|thinking process)` must hit at least once AND response must contain at least 2 numbered steps (`^\s*(1|2|3|4)[.)]`). Requires ≥2 phases of the 4-phase template.
- **A3** — Response-prefix length (300 chars) in sibling data is sufficient to detect method invocation in the opening of the `<|channel>thought` block, since the trained template front-loads "Restate / Identify" within the first 250 chars.
- **A4** — MMLU (cais/mmlu) at n=20 is a noisy but unbiased proxy for MMLU-Pro knowledge; a 30pp drop is far outside the binomial 95% CI (≈21pp at p=0.9, n=20) and therefore a real effect.
- **A5** — Smoke-scale training (N_STEPS=60, n_train=25) is the budget we test; full-scale rerun is deferred to `exp_prompt_erasure_gemma4_v2` if the smoke result is ambiguous.
