# Peer Review: huffman_macro_skew

## NotebookLM Findings

Skipped -- the experiment is primarily analytical (no novel math derivations to stress-test), and the core claims can be verified directly from Shannon coding theory.

## Mathematical Soundness

### What holds

1. **Shannon bound application is correct.** The bound H(f) <= E[d] < H(f) + 1 for Huffman codes is textbook. The derivation that H_norm > 0.95 implies R_max < 5% (Section 4.2 of MATH.md) follows directly:
   - R_max = (D_bal - H(f)) / D_bal
   - H_norm > 0.95 means H(f) > 0.95 * D_bal (for power-of-2 L where D_bal = log2(L))
   - Therefore R_max < 0.05

2. **The two kill criteria are mathematically redundant** (for power-of-2 L). Kill criterion 1 (H_norm > 0.95) implies kill criterion 2 (R < 5%). The paper acknowledges this linkage in Section 4.2. This is not a flaw -- it means the criteria are consistent and the entropy check is the sharper one.

3. **Gradient flow analysis is correct in form.** The sigmoid chain gradient prod[p_i(1-p_i)] with max 0.25 per gate is standard. The critical depth calculation (log(1e-6)/log(0.09) = 5.7 for sharp gates) is arithmetically correct.

4. **Zipf distribution implementation is standard.** The (i+1)^(-alpha) form with normalization is equivalent to standard Zipf ranking from 1 to n.

### What does not hold or is misleading

1. **The gradient analysis overstates the problem.** Section 5 assumes all gates along a path are "sharp" (p ~ 0.9) simultaneously. In practice, a Huffman tree trained with the balance-preserving KL loss from the huffman_tree experiment would have gates at varying confidence levels. The worst case (all gates sharp at the same level) is unlikely in a well-trained tree. The paper partially acknowledges this in Limitation 4 ("gates early in training are closer to 0.5"), but still uses the sharp-gate analysis to populate the primary results table (Table in Section 5.3). This makes the gradient concern appear more severe than it is.

2. **The critical boundary formula is ad hoc.** The approximation `alpha_critical(w) ~ 0.6 + 0.4*w` (Section 4.3) is presented as an analytical result but is actually a linear fit to a small number of sensitivity sweep points. It is not derived from the mixture model. This is fine as a heuristic, but the paper does not clearly distinguish it from the derived results.

## Novelty Assessment

**This is not a mechanism experiment.** Unlike the preceding huffman_tree experiment (which validated a novel application of Huffman coding to MoE routing), this experiment is a feasibility analysis. It asks: "Given that the mechanism works, under what conditions does it help?"

**No novel mechanism is introduced.** The contribution is:
- A parametric distribution model (Zipf + uniform mixture) for MoE expert utilization
- A systematic sweep of when Huffman helps vs. does not

**Prior art check:** The Zipf + uniform mixture model for MoE routing distributions does not appear to have direct precedent in the MoE literature. Most MoE papers report aggregate load statistics (tokens per expert) but do not model the distribution parametrically. The mixture model is a reasonable first-order approximation, and the paper is honest that it is a model, not ground truth.

**No references in REFERENCES.yml for Huffman.** This is consistent -- the experiment does not introduce a new mechanism requiring reference validation.

## Experimental Design

### Strengths

1. **The experiment asks the right question.** The huffman_tree experiment showed the mechanism works but could not test whether real MoE distributions are skewed enough. This experiment addresses that gap directly.

2. **The sensitivity sweep is well-designed.** Sweeping both alpha (natural skew) and w (balance loss strength) across multiple expert counts produces a clear picture of the viability boundary.

3. **The micro empirical component is honest.** Running the actual trained model on micro data and finding H/Hmax = 0.997 (killed) is expected, explicitly stated as expected, and does not contaminate the analytical findings.

4. **Kill criteria are applied uniformly.** All 10 production scenarios are evaluated against both criteria without cherry-picking survivors.

### Weaknesses

1. **The "production scenarios" are entirely synthetic.** The experiment title says "macro scale with natural routing skew" but no natural routing data is used. Every distribution is generated by the mixture_zipf model with researcher-chosen (alpha, w) parameters. The parameter choices for each production system (e.g., DeepSeek-V3 "heavy" = alpha=1.0, w=0.1) are plausible but unvalidated guesses. The paper acknowledges this (Limitation 1, Limitation 3, and the "definitive test" section), but the framing in the results table -- labeling scenarios as "DeepSeek-V3" rather than "Zipf(1.0)+Uniform(0.1) at L=256" -- risks implying empirical grounding that does not exist.

2. **The scenario selection is asymmetric.** Three scenarios per system, but the "heavy skew" option is always the most extreme the researcher modeled. The 3/10 survivors are all extreme-skew scenarios. If the researcher had added a fourth "extreme" scenario to each system (alpha=1.5, w=0.0), 4/14 or more would survive, making the result appear more favorable. Conversely, adding more moderate scenarios would push the kill rate higher. The 7/10 vs 3/10 split is an artifact of which scenarios the researcher chose to model. **The ratio is not meaningful.**

3. **Top-k routing is acknowledged but not modeled.** The mixture model treats each expert's utilization as independent. With top-k routing (k=8 for DeepSeek-V3, k=10 for Qwen3), expert utilizations are correlated: if expert A is selected, expert B's probability changes. This correlation could either increase or decrease effective skew. Without modeling this, the analysis has an unquantified error bar. The paper correctly flags this in Assumption 3 but draws conclusions that ignore it.

4. **No comparison to alternative routing optimizations.** Even if Huffman provides 18% depth reduction, what about simpler approaches? For instance, a two-level routing scheme (first select a cluster of experts, then pick within cluster) achieves O(sqrt(L)) routing without requiring frequency profiling. The experiment does not establish that Huffman is the right tool even when skew exists.

### Controls

The uniform baseline (killed) serves as a negative control. The pure Zipf baselines at various alpha serve as positive controls. These are adequate for the analytical claims.

## Macro-Scale Risks (advisory)

1. **The entire thesis rests on one unknown quantity: H_norm of a real trained MoE.** The paper says this clearly. Until someone profiles DeepSeek-V3, Qwen3, or similar, the conditional pass cannot become unconditional. This is the single highest-value experiment to run at macro scale.

2. **Balance loss dynamics are not static.** During training, balance loss effectiveness changes. Early in training, experts may be more uniform (random routing); later, specialization emerges. The relevant distribution is the converged one, not the training trajectory. This is not addressed.

3. **Frequency stationarity.** Different input domains at inference time will produce different expert utilization distributions. A Huffman tree optimized for one distribution may be suboptimal for another. The paper mentions this but does not quantify the penalty of distribution mismatch.

4. **Implementation overhead.** Even if Huffman saves 1-2 gate evaluations per token on average, the irregular tree structure complicates GPU parallelism. Balanced trees map cleanly to batch operations; Huffman trees require variable-length paths per token. This practical concern could negate the theoretical depth savings.

## Verdict

**PROCEED**

The conditional pass is the correct verdict for the evidence presented. Justification:

1. **The math is sound.** Shannon bounds, entropy calculations, and kill criterion derivations are all correct. No hidden mathematical errors.

2. **The experimental design is appropriate for an analytical study.** The researcher correctly identifies that the definitive test requires profiling a real MoE model, and frames all results as conditional on that measurement.

3. **The 3/10 surviving scenarios are not cherry-picked.** They represent a specific and clearly-delineated regime (heavy natural skew with weak balance loss). The boundary is sharp and well-characterized.

4. **The experiment advances the vision.** Cheap routing is central to VISION.md ("inference selects only the relevant experts per input"). Understanding when Huffman helps vs. when it does not is necessary information for the macro architecture.

5. **The framing is honest.** The paper does not overclaim. "CONDITIONAL PASS" with explicit conditions for validation or killing is exactly the right conclusion.

**Minor issues to note (not blocking):**

1. Update HYPOTHESES.yml: set experiment_dir, add evidence, change status to "conditional" or similar.
2. The 7/10 vs 3/10 ratio should not be quoted as a precise finding -- it depends on which scenarios were chosen. The meaningful finding is the boundary characterization (alpha >= 0.7 with w <= 0.1, or alpha >= 1.0 with w <= 0.5 for L >= 64).
3. The gradient analysis table should note it presents worst-case (all-sharp-gates) numbers. Average-case gradients along Huffman paths would be substantially healthier.
